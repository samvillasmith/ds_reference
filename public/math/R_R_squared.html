What R² Measures
R² answers: "How much better is my regression line than just guessing the average every time?"

You have two ways to "predict" a value:

Dumb model: always guess the average (ȳ) → Ignores x entirely. Just says "the average price is $4k" for every house. → The errors from this model = SStotal   Your model: use the regression line (ŷ) → Uses x to make a prediction. h(x) = θ₀ + θ₁x → The errors from this model = SSres
R² = 1 − SSres / SStotal   = 1 − (your model's errors) / (dumb model's errors)   If your model is perfect → SSres = 0 → R² = 1 If your model = just the average → SSres = SStotal → R² = 0 If your model is worse than the average → R² < 0 (yes, it can go negative)

SStotal — How Bad is the "Dumb Model" (Just Guess the Average)?
SStotal = Σ(yᵢ − ȳ)² — square the distance from each actual value to the average.

The average is ȳ = 4.0. How far is each house from that?

For each house: (actual price − average)²   House 1: (2 − 4.0)² = (−2.0)² = 4.00 House 2: (4 − 4.0)² = (0.0)² = 0.00 House 3: (5 − 4.0)² = (1.0)² = 1.00 House 4: (4 − 4.0)² = (0.0)² = 0.00 House 5: (5 − 4.0)² = (1.0)² = 1.00
SStotal = 4.00 + 0.00 + 1.00 + 0.00 + 1.00 = 6.00

This is the total variance in your data. It's the baseline — if your regression line can't beat this, it's useless.

SSres — How Bad is Your Regression Line?
SSres = Σ(yᵢ − ŷᵢ)² — square the distance from each actual value to the prediction.

First, get the predictions from h(x) = 2.2 + 0.6x:

Predictions:   House 1: ŷ = 2.2 + 0.6 × 1 = 2.8 House 2: ŷ = 2.2 + 0.6 × 2 = 3.4 House 3: ŷ = 2.2 + 0.6 × 3 = 4.0 House 4: ŷ = 2.2 + 0.6 × 4 = 4.6 House 5: ŷ = 2.2 + 0.6 × 5 = 5.2

Now compute (actual − predicted)² for each:

Squared residuals:   House 1: (2 − 2.8)² = (−0.8)² = 0.64 House 2: (4 − 3.4)² = (+0.6)² = 0.36 House 3: (5 − 4.0)² = (+1.0)² = 1.00 House 4: (4 − 4.6)² = (−0.6)² = 0.36 House 5: (5 − 5.2)² = (−0.2)² = 0.04
SSres = 0.64 + 0.36 + 1.00 + 0.36 + 0.04 = 2.40
Compare: SStotal = 6.00 vs SSres = 2.40 Your line's errors (2.40) are less than the average's errors (6.00) — good sign!
R² — Plug It In
R² = 1 − SSres / SStotal   R² = 1 − 2.40 / 6.00   R² = 1 − 0.40   R² = 0.60

How to read this:
R² = 0.60 means your regression line explains 60% of the variance in house prices.
The remaining 40% is unexplained — noise, missing features, randomness.
Another way to think about it:   Your model reduced the total error from 6.00 down to 2.40 That's a 60% reduction — so R² = 0.60
But R² Has a Problem...
R² always goes up (or stays the same) when you add more features — even useless ones.

Add "house color" as a feature? R² goes up.
Add "owner's shoe size"? R² goes up.
Add random noise? R² still goes up.

It can never go down because more parameters = more freedom to fit the data.
This doesn't mean the model is actually better — it might be overfitting.

This is why Adjusted R² exists →

Adjusted R² — Penalizes Extra Features
Adjusted R² adds a penalty for each feature you add. If a new feature doesn't improve the model enough to justify its existence, Adjusted R² goes down.
Adjusted R² = 1 − [(1 − R²) × (n − 1) / (n − p − 1)]   where: n = number of data points p = number of features R² = the regular R² we just computed
Example 1: Simple Regression (1 feature)

h(x) = 2.2 + 0.6x — just using sq ft

n = 5 (data points)    p = 1 (features)    R² = 0.60   Adj R² = 1 − [( 1 − 0.60 ) × ( 5 − 1 ) / ( 5 − 1 − 1 )]   = 1 − [ 0.40 × 4 / 3 ]   = 1 − [ 1.60 / 3 ]   = 1 − 0.5333   = 0.4667
R² = 0.60 → Adjusted R² = 0.4667    (lower, because small dataset)
Example 2: What If We Add a Useless Feature?

Say we add "owner's shoe size" and R² barely increases to 0.61

n = 5    p = 2 (now 2 features)    R² = 0.61   Adj R² = 1 − [( 1 − 0.61 ) × ( 5 − 1 ) / ( 5 − 2 − 1 )]   = 1 − [ 0.39 × 4 / 2 ]   = 1 − [ 1.56 / 2 ]   = 1 − 0.78   = 0.22
R² went up: 0.60 → 0.61    but Adjusted R² went DOWN: 0.4667 → 0.22
R² said "adding shoe size helped."
Adjusted R² said "no it didn't — you just added noise."

The penalty kicked in because p went from 1 to 2, but R² barely moved. The denominator (n − p − 1) shrank from 3 to 2, amplifying the penalty.
Example 3: What If We Add a Useful Feature?

Say we add "bedrooms" and R² jumps to 0.85

n = 5    p = 2    R² = 0.85   Adj R² = 1 − [( 1 − 0.85 ) × ( 5 − 1 ) / ( 5 − 2 − 1 )]   = 1 − [ 0.15 × 4 / 2 ]   = 1 − 0.30   = 0.70
R² went up: 0.60 → 0.85    AND Adjusted R² went UP: 0.4667 → 0.70 ✅
Both metrics agree — bedrooms is a genuinely useful feature. The R² improvement was large enough to overcome the penalty.
Summary
When to use which:   R² — fine when comparing models with the same number of features   Adjusted R² — use when comparing models with different numbers of features It tells you if that extra feature is actually earning its keep   Rule of thumb: if Adjusted R² drops when you add a feature, remove that feature.
