Simple vs Multiple — What Changed? Simple vs Múltiple — ¿Qué Cambió? 単回帰 vs 重回帰 — 何が変わったか？
The entire algorithm is identical. The only difference is you repeat the θ update step once per feature.
El algoritmo completo es idéntico. La única diferencia es que repites el paso de actualización de θ una vez por cada característica.
アルゴリズム全体は同一です。唯一の違いは、特徴量ごとに θ の更新ステップを繰り返すことです。
Simple (1 feature): Simple (1 característica): 単純 (1特徴量):   h(x) = θ₀ + θ₁x   Update θ₀ → errors × 1Actualizar θ₀ → errores × 1θ₀ を更新 → 誤差 × 1 Update θ₁ → errors × xActualizar θ₁ → errores × xθ₁ を更新 → 誤差 × x
Multiple (2 features): Múltiple (2 características): 重回帰 (2特徴量):   h(x) = θ₀ + θ₁x₁ + θ₂x₂   Update θ₀ → errors × 1      ← sameActualizar θ₀ → errores × 1  ← igualθ₀ を更新 → 誤差 × 1  ← 同じ Update θ₁ → errors × x₁    ← sameActualizar θ₁ → errores × x₁  ← igualθ₁ を更新 → 誤差 × x₁  ← 同じ Update θ₂ → errors × x₂    ← new, same patternActualizar θ₂ → errores × x₂  ← nuevo, mismo patrónθ₂ を更新 → 誤差 × x₂  ← 新規、同じパターン

The general update rule for any θⱼ: La regla general de actualización para cualquier θⱼ: 任意の θⱼ に対する一般的な更新規則:
θⱼ := θⱼ − α · (1/m) · Σ( h(x⁽ⁱ⁾) − y⁽ⁱ⁾ ) · xⱼ⁽ⁱ⁾   where xⱼ is the j-th featuredonde xⱼ es la j-ésima característicaここで xⱼ は j 番目の特徴量 and x₀ = 1 always (that's why θ₀ multiplies by 1)y x₀ = 1 siempre (por eso θ₀ multiplica por 1)そして x₀ = 1 は常に成り立つ (だから θ₀ は1を掛ける)
Bottom line: En resumen: 要点:   Adding more features doesn't change the algorithm.Añadir más características no cambia el algoritmo.特徴量を増やしてもアルゴリズムは変わりません。 It just adds more θ values to update each iteration.Solo añade más valores de θ para actualizar en cada iteración.各反復で更新する θ の値が増えるだけです。 Each θ update uses the exact same formula.Cada actualización de θ usa exactamente la misma fórmula.各 θ の更新は全く同じ式を使います。
