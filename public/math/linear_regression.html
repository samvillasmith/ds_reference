Simple vs Multiple — What Changed?
The entire algorithm is identical. The only difference is you repeat the θ update step once per feature.
Simple (1 feature):   h(x) = θ₀ + θ₁x   Update θ₀ → errors × 1 Update θ₁ → errors × x
Multiple (2 features):   h(x) = θ₀ + θ₁x₁ + θ₂x₂   Update θ₀ → errors × 1      ← same Update θ₁ → errors × x₁    ← same Update θ₂ → errors × x₂    ← new, same pattern
n features? Same thing, just keep going:   h(x) = θ₀ + θ₁x₁ + θ₂x₂ + θ₃x₃ + ... + θₙxₙ   Update θ₀ → errors × 1 Update θ₁ → errors × x₁ Update θ₂ → errors × x₂ Update θ₃ → errors × x₃   ⋮ Update θₙ → errors × xₙ
The general update rule for any θⱼ:
θⱼ := θⱼ − α · (1/m) · Σ( h(x⁽ⁱ⁾) − y⁽ⁱ⁾ ) · xⱼ⁽ⁱ⁾   where xⱼ is the j-th feature and x₀ = 1 always (that's why θ₀ multiplies by 1)
Bottom line:   Adding more features doesn't change the algorithm. It just adds more θ values to update each iteration. Each θ update uses the exact same formula.
