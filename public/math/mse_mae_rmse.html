2 2 4 3.4 +0.6
3 3 5 4.0 +1.0
4 4 4 4.6 −0.6
5 5 5 5.2 −0.2
How predictions were computed:   House 1: ŷ = 2.2 + 0.6 × 1 = 2.8 House 2: ŷ = 2.2 + 0.6 × 2 = 3.4 House 3: ŷ = 2.2 + 0.6 × 3 = 4.0 House 4: ŷ = 2.2 + 0.6 × 4 = 4.6 House 5: ŷ = 2.2 + 0.6 × 5 = 5.2
The problem with raw errors: they cancel out.

−0.8 + 0.6 + 1.0 + −0.6 + −0.2 = 0.0

The negatives and positives cancel perfectly. Sum = 0. That doesn't mean the model is perfect — it means raw errors are useless as a metric. We need to get rid of the signs.
Two ways to kill the sign:   ① Take absolute value → |error| → that's MAE ② Square the error → error² → that's MSE
MAE — Mean Absolute Error
MAE = (1/n) × Σ|yᵢ − ŷᵢ|

Take each error, drop the sign (absolute value), average them. Done.
Step 1 — Take absolute value of each error:   House 1: |−0.8| = 0.8 House 2: |+0.6| = 0.6 House 3: |+1.0| = 1.0 House 4: |−0.6| = 0.6 House 5: |−0.2| = 0.2
Step 2 — Average them:   MAE = ( 0.8 + 0.6 + 1.0 + 0.6 + 0.2 ) / 5   = 3.2 / 5   = 0.64
MAE = 0.64 → "On average, predictions are off by $0.64k ($640)"
Why use MAE?   ✓ Easy to understand — it's in the same units as y ($k) ✓ Treats all errors equally — a $1k miss counts the same everywhere ✓ Robust to outliers — one huge error doesn't dominate ✗ Not differentiable at 0 — harder to use in gradient descent
MSE — Mean Squared Error
MSE = (1/n) × Σ(yᵢ − ŷᵢ)²

Take each error, square it, average them. Squaring kills the sign AND punishes big errors more.
Step 1 — Square each error:   House 1: (−0.8)² = 0.64 House 2: (+0.6)² = 0.36 House 3: (+1.0)² = 1.00 House 4: (−0.6)² = 0.36 House 5: (−0.2)² = 0.04
Step 2 — Average them:   MSE = ( 0.64 + 0.36 + 1.00 + 0.36 + 0.04 ) / 5   = 2.40 / 5   = 0.48
MSE = 0.48 → "Average squared error is 0.48"
Notice the units problem:

Your y is in $k. You squared the errors, so MSE is in $k².
What's "$0.48 thousand-dollars-squared"? It doesn't mean anything intuitive.

This is why RMSE exists →
Why use MSE?   ✓ Differentiable everywhere — perfect for gradient descent ✓ Punishes large errors more — a 2.0 error costs 4× more than a 1.0 error ✓ This is what the cost function J(θ) is based on ✗ Units are squared — not directly interpretable ✗ Sensitive to outliers — one huge error can dominate
How squaring punishes big errors:   Error of 0.2 → squared = 0.04  (tiny) Error of 1.0 → squared = 1.00  (25× bigger!) Error of 5.0 → squared = 25.0  (625× bigger!)   The bigger the miss, the more MSE cares about it. MAE would treat these proportionally: 0.2, 1.0, 5.0.
RMSE — Root Mean Squared Error
RMSE = √MSE

That's it. Take MSE and square-root it. This undoes the squaring so units make sense again.
Just one step — take the square root of MSE:   RMSE = √MSE   = √0.48   = 0.6928
RMSE = 0.6928 → "Predictions are off by ~$0.69k ($693) on average"
Or expand the full formula:   RMSE = √[ (1/n) × Σ(yᵢ − ŷᵢ)² ]   = √[ (0.64 + 0.36 + 1.00 + 0.36 + 0.04) / 5 ]   = √[ 2.40 / 5 ]   = √0.48   = 0.6928
Why use RMSE?   ✓ Same units as y ($k) — directly interpretable ✓ Still punishes big errors more (from the squaring inside) ✓ Most commonly reported metric in ML papers ✗ Still sensitive to outliers (like MSE)

Why is RMSE > MAE?   RMSE is always ≥ MAE. The squaring step amplifies larger errors, so the average gets pulled up. The bigger the gap between RMSE and MAE, the more variance in your errors (some big, some small).   If RMSE ≈ MAE → errors are all similar sizes If RMSE >> MAE → a few big errors are dragging it up
How They're Related
The chain:   MAE = avg of |errors|   → simple average MSE = avg of errors²   → square first, then average RMSE = √(avg of errors²) → square, average, then undo the square   MSE and RMSE contain the same information — RMSE is just MSE in human-readable units.
When to Use Which
Use MAE when: • You want a simple "how far off am I on average?" answer • You have outliers and don't want them to dominate • All errors matter equally regardless of size • Example: predicting delivery time — off by 5 min is 5× worse than 1 min   Use MSE when: • You're training a model (it's the cost function) • You need smooth gradients for optimization • Big errors should be punished disproportionately   Use RMSE when: • You want MSE's properties but in interpretable units • Reporting results to someone who needs to understand them • Example: "our model is off by ~$693 on average" (RMSE)   vs "the mean squared error is 0.48" (MSE) — which is clearer?
