Ordinary Least Squares

The workhorse of regression — and the foundation on which much of modern data science is built.

Definition

What Is OLS?

Ordinary Least Squares (OLS) is a method for estimating the parameters of a linear regression model. Given a set of data points, OLS finds the line (or hyperplane) that minimises the sum of the squared differences between the observed values and the values predicted by the model.

In plain language: you have data, you want the "best" straight line through it, and OLS tells you exactly where that line goes — by making the total squared error as small as possible.

ŷ = β₀ + β₁x₁ + β₂x₂ + … + βₚxₚ

The linear model: ŷ is the prediction, β₀ is the intercept, and β₁…βₚ are the coefficients for each feature.

minimise   Σ(yᵢ − ŷᵢ)²

The OLS objective: choose β values to minimise the sum of squared residuals.

Visual Intuition

How It Works

Each vertical dashed line below is a residual — the distance between an observed data point and the fitted line. OLS finds the line that makes the total area of the shaded squares as small as possible.

The OLS regression line minimises the total area of the shaded squares — hence "least squares."

Assumptions

The Five Key Assumptions

OLS produces the Best Linear Unbiased Estimator (BLUE) — but only when these assumptions hold.

#AssumptionWhat It MeansWhat Breaks If Violated
1LinearityThe relationship between features and target is linear.Biased estimates — the model systematically misses the true pattern.
2IndependenceObservations are independent of each other.Standard errors are wrong; hypothesis tests become unreliable.
3HomoscedasticityResiduals have constant variance across all levels of X.Estimates still unbiased but inefficient; confidence intervals misleading.
4Normality of residualsResiduals are normally distributed (needed for valid inference).p-values and confidence intervals lose validity, especially in small samples.
5No multicollinearityFeatures are not highly correlated with each other.Coefficients become unstable; standard errors inflate.
Practical tip: In the real world, these assumptions are almost never perfectly met. The key is understanding which is violated and how badly — mild violations often don't matter, while severe ones require alternative methods.
Applications

What Is OLS Used For?

Explanation & Inference

  • Understanding which features drive an outcome (and by how much)
  • A/B test analysis — estimating treatment effects with covariates
  • Econometric modelling — causal inference, policy evaluation
  • Scientific research — testing hypotheses about relationships

Prediction & Baselines

  • Quick baseline model before trying complex algorithms
  • Forecasting when the relationship is approximately linear
  • Feature selection — coefficients reveal which inputs matter
  • Foundation for Ridge, Lasso, Elastic Net

Interpreting the Output

Each coefficient β tells you: "Holding all other features constant, a one-unit increase in this feature is associated with a β-unit change in the target." This makes OLS the default when stakeholders need to understand why a model makes a prediction.

Key Diagnostics

  • R² — proportion of variance explained. Higher is better, but a high R² alone doesn't validate the model.
  • Adjusted R² — penalises for adding features, preventing artificial inflation.
  • p-values — probability of seeing a coefficient this extreme if the true value were zero.
  • F-statistic — tests whether the model as a whole is significant.
  • Residual plots — visual checks for linearity, homoscedasticity, and normality.
Tradeoffs

Strengths & Limitations

Strengths

  • Closed-form solution — no iterative tuning
  • Highly interpretable coefficients
  • Fast to fit, even on large datasets
  • Well-understood statistical properties
  • Foundation for many advanced methods

Limitations

  • Assumes linear relationships — misses curves and interactions
  • Sensitive to outliers
  • Struggles with multicollinearity
  • Can overfit when p > n
  • Predictions aren't bounded

Bottom line: OLS isn't the most powerful predictive tool today, but it's one of the most important things to understand deeply. It builds the intuition — about residuals, bias, variance, regularisation — that makes everything else click.
Code

Quick Reference: Python

# Using scikit-learn
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.metrics import r2_score, mean_squared_error

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

model = LinearRegression()
model.fit(X_train, y_train)

y_pred = model.predict(X_test)

print(f"R²:   {r2_score(y_test, y_pred):.4f}")
print(f"RMSE: {mean_squared_error(y_test, y_pred, squared=False):.4f}")
print(f"Coefficients: {model.coef_}")
print(f"Intercept:    {model.intercept_}")

For full statistical inference (p-values, confidence intervals), use statsmodels.OLS instead.
